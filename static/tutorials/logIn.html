Hackaton Security API

The simplified hackaton security API allows to directly register clients and users, log them in and out and let them agree to applications (without UI interaction or permissions). We expect this to simplify the overall interaction during the hackaton.

In order to utilize the OPENi system a client and user are required. Examples of these commands are shown below.

Security API UI

To get to the Security API UI from the home page click the "Security API" link within the "Client Libraries" section, as shown below.

Client Registration

This endpoint allows you to register new clients. The client registration requires a client_id. If not existent, a new client with the id will be created.

Upon success this call will return an empty responce with a 200 status code.

Endpoint


POST /uaa/clients

Request


{"client_id": ""}

User Registration

This endpoint allows you to register new users. The user registration requires a username, an email and a password. If not existent, a new user will be created.

Upon success this call will return an empty responce with a 200 status code.

Endpoint


POST /uaa/users

Request


{"name": "", "password": ""}

User Login

This endpoint allows you to log in existing users. It requires a username and a password. If the user does not exist, the call will fail. The login will return a session. The session will expire eventually.

Upon success this call will return a responce object, as seen below and with a 200 status code.

Endpoint


POST /uaa/session

Request


{"name": "", "password": ""}

Response


{"session": ""}

User Authorizes App

This endpoint allows you to authorize clients on behalf of a user. It requires a session and a client. If the user or the client do not exist, the call will fail. The authorization will return a token that can be passed to the resource platforms.

This request takes a previously queried session key and the client_id from the applications the user is using to access the platform.

Upon success this call will return a responce object with a token, as seen below and with a 200 status code.

This token is then used within the **Cloudlet API** to access the cloudlet of the owner of the *token*.

Endpoint


POST /uaa/authorize

Request


{"session": "", "client_id": ""}

Response


{"token": ""}

User Logout

This endpoint allows you to log out a user, if so needed.

Upon success this call will return an empty responce with a 200 status code.

Endpoint


DELETE /uaa/session

Request


{"session": ""}
